# Contributing to Peggy

You don't need to install anything or use the command line. GitHub's web editor is the easiest way to contribute.

## Editing an existing doc

1. Go to [github.com/MrDingle-png/peggy](https://github.com/MrDingle-png/peggy)
2. Navigate to the file you want to edit inside `Knowledge/`
3. Click the **pencil icon** (top right of the file view) to open the editor
4. Make your changes — the files are Markdown, so you can use `#` for headings, `-` for bullet points, `**bold**` for bold, etc.
5. Click **Commit changes...**
6. Add a short description of what you changed
7. Select **Create a new branch for this commit and start a pull request**
8. Click **Propose changes**, then **Create pull request**

Someone will review and merge your changes.

## Adding a new doc

1. Navigate to the folder where the doc belongs (e.g. `Knowledge/Quests/`)
2. Click **Add file** > **Create new file**
3. Name your file using kebab-case with a `.md` extension (e.g. `my-new-strategy.md`)
4. Write your content
5. Follow steps 5-8 above to submit it as a pull request

## Which folder should I use?

| Folder | Use for |
|--------|---------|
| `Knowledge/Quests/` | Quest strategy, quest frameworks, quest PRDs |
| `Knowledge/Games & Interactive/` | Games strategy, app architecture, structural model |
| `Knowledge/Programs & Quests/` | Shared building blocks across programmes and quests |
| `Knowledge/FanCash Trading Card/` | FanCash proposals, redemption specs, flows |
| `Knowledge/Instant Rips/` | Digital packs, instant rips PRDs |

If your doc doesn't fit an existing folder, put it at the `Knowledge/` root or suggest a new folder in your pull request.

## What belongs here

Peggy is for **product knowledge only** — strategy, frameworks, PRDs, and product analysis for Fanatics Collect. Operational docs (project plans, hiring proposals, team processes) live elsewhere.

## Markdown basics

```
# Heading 1
## Heading 2
### Heading 3

**bold text**
*italic text*

- bullet point
- another bullet

1. numbered item
2. another item

> blockquote

[link text](https://url.com)
```
